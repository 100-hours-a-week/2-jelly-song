# 간략 회고

### 우아한 레디스 컨퍼런스

우아한 레디스 라는 유튜브 컨퍼런스가 있어 해당 영상을 시청하였다. 원래 과거에 한번 시청했었던 것인데, 샤딩이라던지 이런 부분에 있어 기본적인 부분을

잘 다루어주었던 강의로 기억에 남아 다시 시청 하였다.

가장 인상 깊었던 부분은 샤딩을 할 때, 어찌 쪼갤 수 있는지에 대한 방법이었다. 크게 모듈러 방법과 레인지 방법으로 나누어서 설명을 해주셨는데

모듈러 방법의 경우 만약 db 서버가 하나 더 추가된다면 그 안에 있는 데이터들의 이동이 많이 일어나는 것을 확인할 수 있었다. 그러나 레인지 방법은

그냥 레인지 범위를 추가하고 늘리면 되기 때문에 간단하였다. 레인지의 경우에는 레인지를 기준으로 한번에 흝을 때 한 서버에 있기 때문에 바로바로 찾을 수 있다는

장점이 있고, 단점으로는 특정 특성을 가진 데이터가 어떤 레인지에 몰려, 해당 서버에 부하가 없거나 부하가 많은 문제가 발생할 수 있다.

해시는 어떤 모듈러 값을 기준으로 나누기 때문에 적절히 분산될 수 있다는 장점이 있다. 다만 아까 본 예시와 같이 이동이 많이 일어나게 되는데 이것은

Consistent Hashing 로 보완이 가능하다. 해당 해시 값이 특정 서버의 해시 값보다 가우스 기호를 붙여서 넣는 방식이다. 이렇게 된다면 서버가 추가되거나

사라진다 하더라도 해당 서버에만 이동이 일어나기 때문에 이를 방지할 수 있다.

이번 컨퍼런스를 보고, 대규모 환경에서는 정말 고려할게 많다고 생각하였다. 디비 지식이나 이런 샤딩, 레플리카 같은 지식을 함양하려고 꾸준히 노력해야겠다는

생각을 하였다.
