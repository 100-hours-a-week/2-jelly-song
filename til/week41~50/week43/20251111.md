# 간략 회고

### 대규모 트래픽 캐시 전략

대규모 트래픽 캐시 전략 이론 부분을 수강 완료하였다.

이 강의를 들은 이유는 저번 프로젝트에서 캐시 스탬피드를 막기 위해 PER 알고리즘을 적용하고, 캐시 페네트레이션을 막기 위해

널 캐싱을 활용했었다. 이런 경험을 바탕으로 이 강의를 추가로 듣는다면 다양한 방법론을 익힐 수 있을 것 같아 수강하였다.

캐시 페네트레이션을 막는 방법은 null 캐싱 말고도 블룸 필터 방법이 있었다. 블룸 필터 방법은 비트를 이용하여 데아터의 존재 유무를 판단하는 방법이다.

해당 데이터를 비트로 표시하여 캐시 페네트레이션을 막는 방법이다. 널 캐싱 보다 비트로 관리하여 메모리 효율적이라는

장점이 있다. 단, 한번 정한 비트 수는 바꿀 수 있다는 단점이 있다.

캐시 스탬피드를 막기 위한 방법으로는 추가적으로 Jitter와 Request Collapsing, Rate Limit이 있었다.

Jitter의 경우에는 이미 알고 있던 방법이지만 캐싱 Key 마다 랜덤한 값을 + -로 두어 캐시 만료 시간을 다르게 하는 방법이다.

Request Collapsing의 경우에는 캐시 만료가 발생하여 보조기억장치로의 접근이 필요할 때 모두 접근하는 것이 아니라 하나의 요청만

가도록 설정하는 방법이다. Reat Limit의 경우에는 만약 캐시가 있을 경우 2000요청까지 어플리케이션이 버틸 수 있는데,

데이터 베이스는 1000요청 까지 버틸 수 있다고 가정했을때, 캐시가 만료되어 데이터베이스로 2000요청이 가면 다운될 수 있으므로 그

구간의 요청 한계를 1000으로 맞추는 방법을 뜻한다.
